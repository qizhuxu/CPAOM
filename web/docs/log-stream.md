# 系统日志流功能

## 功能概述

Web 版 CPA 管理系统现在支持实时查看后端日志，方便调试和监控系统运行状态。

## 功能特性

### 1. 实时日志流
- 使用 Server-Sent Events (SSE) 技术实现实时推送
- 自动连接和重连机制
- 心跳保持连接活跃

### 2. 日志过滤
- **级别过滤**: DEBUG, INFO, WARNING, ERROR, CRITICAL
- **内容搜索**: 支持实时搜索日志内容
- **组合过滤**: 可同时使用级别和搜索过滤

### 3. 日志控制
- **暂停/继续**: 暂停日志流更新，方便查看
- **自动滚动**: 自动滚动到最新日志（可关闭）
- **清空显示**: 清空当前显示的日志
- **下载日志**: 将日志导出为文本文件

### 4. 日志显示
- **彩色显示**: 不同级别使用不同颜色
  - DEBUG: 灰色
  - INFO: 青色
  - WARNING: 黄色
  - ERROR: 红色
  - CRITICAL: 亮红色
- **格式化**: 时间戳 + 级别 + 模块名 + 消息
- **缓冲区**: 保存最近 1000 条日志

## 使用方法

### 访问日志页面

1. 登录系统
2. 点击侧边栏的 "系统日志" 菜单
3. 日志流会自动连接并开始显示

### 过滤日志

**按级别过滤:**
```
1. 选择日志级别下拉框
2. 选择要查看的级别（如 ERROR）
3. 只显示该级别的日志
```

**按内容搜索:**
```
1. 在搜索框输入关键词
2. 实时过滤包含关键词的日志
3. 支持模糊匹配
```

### 控制日志流

**暂停日志:**
- 点击 "暂停" 按钮
- 日志流停止更新，但连接保持
- 再次点击 "继续" 恢复更新

**清空显示:**
- 点击 "清空显示" 按钮
- 清空当前显示的所有日志
- 不影响后端日志缓冲区

**下载日志:**
- 点击 "下载日志" 按钮
- 将当前显示的日志导出为 .txt 文件
- 文件名格式: `cpa-logs-YYYY-MM-DD-HH-MM-SS.txt`

## API 端点

### 1. 日志流 (SSE)
```
GET /api/logs/stream
```
- 返回实时日志流
- 使用 EventSource 连接
- 自动发送历史日志

### 2. 历史日志
```
GET /api/logs/history?limit=100&level=INFO
```
- 获取历史日志
- 参数:
  - `limit`: 返回数量（默认 100）
  - `level`: 日志级别过滤（可选）

### 3. 清空日志
```
POST /api/logs/clear
```
- 清空后端日志缓冲区
- 需要登录权限

```python
    - 维护日志缓冲区（1000条）
    - 通知所有订阅者
```

```python

def stream_logs():
    - 使用 Flask Response + stream_with_context

    - 持续推送新日志
```


## 日志级别说明

| 级别 | 用途 | 示例 |
|------|------|------|
| DEBUG | 调试信息 | 变量值、函数调用 |
| INFO | 一般信息 | 操作成功、状态变化 |
| WARNING | 警告信息 | 可恢复的错误、弃用警告 |
| ERROR | 错误信息 | 操作失败、异常捕获 |
| CRITICAL | 严重错误 | 系统崩溃、数据丢失 |

## 性能考虑

1. **日志缓冲区限制**: 最多保存 1000 条日志，避免内存溢出
2. **前端显示限制**: 前端也限制 1000 条，超出自动删除旧日志
3. **连接超时**: 30 秒无日志时发送心跳保持连接
4. **自动重连**: 连接断开后 5 秒自动重连

## 注意事项

1. **敏感信息**: 日志中不应包含密码、Token 等敏感信息
2. **日志级别**: 生产环境建议使用 INFO 及以上级别
3. **性能影响**: 大量日志可能影响性能，建议适当过滤
4. **浏览器兼容**: 需要支持 EventSource 的现代浏览器

## 故障排除

### 日志流无法连接

**问题**: 显示 "正在连接日志流..." 但一直无法连接

**解决方案**:
1. 检查后端是否正常运行
2. 检查浏览器控制台是否有错误
3. 确认 `/api/logs/stream` 端点可访问
4. 检查防火墙或代理设置

### 日志不更新

**问题**: 日志流已连接但不显示新日志

**解决方案**:
1. 检查是否点击了 "暂停" 按钮
2. 检查日志级别过滤是否过于严格
3. 检查搜索框是否有过滤条件
4. 刷新页面重新连接

### 日志显示乱码

**问题**: 日志中文显示为乱码

**解决方案**:
1. 确保后端使用 UTF-8 编码
2. 检查浏览器编码设置
3. 确认 Flask 响应头包含正确的 charset
