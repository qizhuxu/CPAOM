## 📖 文档维护

### 文档更新原则

1. **与代码同步**: 代码变更时同步更新文档
2. **保持简洁**: 避免冗余信息，重点突出
3. **实例驱动**: 提供具体的使用示例
4. **版本控制**: 重要变更记录在更新日志中

### 文档贡献

欢迎改进文档质量：

1. **发现错误**: 提交 Issue 报告文档错误
2. **改进内容**: 提交 PR 改进文档内容
3. **添加示例**: 补充使用示例和最佳实践
4. **翻译文档**: 提供其他语言版本

### 文档规范

- **Markdown 格式**: 所有文档使用 Markdown 编写
- **中文优先**: 主要文档使用中文，关键术语保留英文
- **链接检查**: 确保所有内部链接有效
- **代码示例**: 提供可运行的代码示例
