# Documentation Management - Reference Index

Quick navigation for documentation management best practices and guidelines.

## Core References

### [mvd-principles.md](mvd-principles.md)
Minimum Viable Documentation principles:
- Focus on WHY over HOW
- Keep it actionable and current
- Capture essential information only
- Avoid over-documentation

### [structure-guide.md](structure-guide.md)
Project documentation structure:
- Root level files (README, LICENSE)
- /docs folder organization
- File naming conventions
- When to split or merge docs

### [writing-guide.md](writing-guide.md)
Technical writing guidelines:
- Clear, concise language
- Code examples best practices
- Formatting and structure
- Common pitfalls to avoid

### [templates.md](templates.md)
Ready-to-use documentation templates:
- README.md template
- Architecture Decision Records (ADR)
- API documentation template
- Development setup guide
- Deployment guide

## Quick Navigation

**Starting a New Project:**
1. Use README template from `templates.md`
2. Follow structure from `structure-guide.md`
3. Apply MVD principles from `mvd-principles.md`

**Improving Existing Docs:**
1. Review against MVD principles
2. Consolidate using structure guide
3. Rewrite using writing guide

**Adding New Features:**
1. Update relevant docs in /docs folder
2. Add one-line mention in README if public-facing
3. Follow writing guide for clarity
